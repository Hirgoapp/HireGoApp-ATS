## 🎓 Key Design Decisions

### Multi-Tenancy
✅ **Decision**: Single database with company_id isolation  
**Why**: Simplified operations, easier data analytics, cost-effective  
**Risk**: Must enforce at application layer (no DB-level trust)  
**Mitigation**: Middleware validates company_id on every request  

### Customization
✅ **Decision**: Metadata-driven (JSONB custom fields)  
**Why**: Support any field without schema changes  
**Risk**: Query complexity increases  
**Mitigation**: Proper indexing and caching  

### Architecture
✅ **Decision**: Service layer with dependency injection  
**Why**: Testable, maintainable, scalable  
**Risk**: More initial setup time  
**Mitigation**: Use NestJS framework with best practices  

### Database
✅ **Decision**: PostgreSQL with TypeORM  
**Why**: Powerful, open-source, JSONB support  
**Risk**: Requires expertise  
**Mitigation**: Comprehensive schema documentation  

### API
✅ **Decision**: RESTful with standard patterns  
**Why**: Simple, well-understood, cacheable  
**Risk**: May need GraphQL later  
**Mitigation**: API versioning from day one
